# /start

## 목적

작업 시작 단계를 수행한다.

## 절차

1. 사용자 요청을 한 문장으로 재정리한다.
2. 영향 영역을 backend / primary_frontend / secondary_app / integration / design / testing / harness 중에서 고른다.
3. `AGENTS.md`, `docs/harness/context/BASELINE.md`, `docs/harness/context/INDEX.md`, `docs/harness/README.md`를 확인한다.
4. 관련 layer/gate 문서와 필요한 세부 컨텍스트 문서를 라우팅한다.
5. 단순하지 않은이면 `/plan`으로 넘어간다.
