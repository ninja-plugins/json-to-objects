# Eval 루프

이 디렉터리는 하네스가 실제 프로젝트에서 효과를 냈는지 확인하기 위한 지표와 회귀 사례를 관리한다.

## 파일

- `metrics.md`: 작업별 정량/정성 지표
- `regression-cases.md`: 실패 사례와 재발 방지 반영 내역

## 운영 원칙

1. completed plan에서 반복되는 실패를 찾는다.
2. 실패를 테스트, 문서, skill, agent 지침 중 하나로 되돌린다.
3. 다음 유사 작업에서 같은 실패가 재발하는지 확인한다.
4. 재발하면 하네스 규칙 또는 프로젝트 게이트를 강화한다.
