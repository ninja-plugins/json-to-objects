# 백엔드 리뷰 기준

백엔드 상세 기준은 `docs/harness/01_BACKEND.md`에 있다. 이 파일은 기존 참조 호환을 위한 얇은 체크리스트다.

## 체크리스트

- 도메인/애플리케이션/표현/인프라 경계가 유지됐다.
- DDD/OOP/SOLID/Clean Code는 `docs/harness/10_BACKEND_QUALITY_GATE.md`로 점검했다.
- 표현 계층 코드가 영속성 세부사항을 노출하지 않는다.
- 트랜잭션과 멱등성 경계가 애플리케이션 서비스 또는 도메인 책임에 맞다.
- 관련이 있으면 프레임워크 트랜잭션 동작, 읽기 전용 동작, propagation, rollback, self-invocation/proxy 제약, 커밋 후 실행, outbox 필요성을 확인했다.
- 권한/리소스 경계와 민감 정보 마스킹을 확인했다.
- 목록 엔드포인트 변경은 페이지네이션 제한, 필터링, 정렬, 빈 페이지 종료 조건을 포함한다.
- 동작 변경에는 RED/GREEN/REFACTOR 증거가 있거나 문서화된 예외가 있다.
- 장기 사실이 바뀌었다면 관련 백엔드/통합 컨텍스트를 갱신했다.

## 실패 예시

- 표현 계층 코드에 핵심 비즈니스 규칙이 있다.
- 트랜잭션 경계가 불명확하거나 실제 적용되지 않는다.
- God Service, setter 남발, 도메인 규칙 분산 때문에 변경 비용이나 결함 위험이 크다.
- 동시성 또는 중복 요청 위험에 테스트와 데이터 제약이 없다.
