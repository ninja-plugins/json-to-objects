# 통합 리뷰 기준

통합 상세 기준은 `docs/harness/04_INTEGRATION.md`에 있다. 이 파일은 기존 참조 호환을 위한 얇은 체크리스트다.

## 체크리스트

- 요청/응답 구조가 소비자 사용과 일치한다.
- 인증, 권한, 리소스 선택, 행위자 소속/소유 경계가 유지된다.
- 페이지네이션/목록 로딩 계약이 안정적이다.
- 행위자나 마스킹 규칙이 다르면 주요/보조/공개/통합 API 영역이 의도적으로 분리된다.
- 소비자용 응답이 다른 화면 영역 전용 필드나 내부 구현 필드를 피한다.
- 필요한 경우 제공자/소비자 짝 변경을 함께 처리했다.
- 동작/계약 변경에는 RED/GREEN/REFACTOR 증거가 있거나 문서화된 예외가 있다.
- 장기 API 사실이 바뀌었다면 `docs/harness/context/integration/api-matrix.md`를 갱신했다.
