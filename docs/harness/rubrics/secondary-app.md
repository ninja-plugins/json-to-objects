# 보조 앱 리뷰 기준

> 프로젝트별 행위자/API/토큰/패키지 값은 `docs/harness/profiles/**`에 둔다.

- 마지막 갱신: 2026-05-22
- 범위: `<secondary-app-dir>`, 보조 프론트엔드, 선택적 모바일/PWA/네이티브 런타임

상세 기준은 `docs/harness/03_SECONDARY_APP.md`에 있다. 이 파일은 기존 참조 호환을 위한 얇은 체크리스트다.

## 제품 경계

- 보조 앱은 주요 프론트엔드의 축소판이 아니라 행위자별 작업 흐름 화면이다.
- 화면은 인증/부트스트랩, 프로필/리소스 컨텍스트, 핵심 작업 흐름, 행동 작업 흐름, 문서/파일, 알림, 런타임 통합 같은 사용자 작업 흐름 단위로 구성한다.
- 주요 운영/관리 흐름은 프로젝트 프로필이 해당 행위자 범위에 포함한다고 명시한 경우에만 노출한다.
- 행위자 라벨, 리소스 용어, 기능 도메인 이름은 프로필/컨텍스트에서 가져온다.

## 아키텍처

- 저장소/작업공간 위치와 런타임 스택은 `context/BASELINE.md`와 `profiles/project-profile.md`를 따른다.
- 밀도, 런타임, 내비게이션, 권한 모델이 다르면 주요 화면 영역의 컴포넌트/스타일을 직접 가져오지 않는다.
- 디자인 추가/리팩토링은 `docs/harness/07_DESIGN_SYSTEM.md`와 디자인 시스템 프로필을 따른다.
- UI는 프로젝트가 소유한 스타일을 사용하며 승인 없는 외부 스타일/UI 프레임워크가 없다.
- 공유 패키지는 API/타입/헬퍼 계약이 안정된 뒤에만 추출한다.
- 네이티브/PWA/런타임 기능은 가능한 경우 브라우저 대체 동작 또는 실패 시 차단 동작을 함께 둔다.

## API 계약

- API prefix와 레거시 허용 목록은 프로필/API matrix 값이다.
- shared 또는 legacy API는 API matrix가 보조 소비자를 허용할 때만 재사용한다.
- 인증 부트스트랩, 리소스 컨텍스트, 선택된 리소스, refresh/logout, 행위자 capability 흐름이 주요 프론트엔드와 충돌하지 않는다.
- 중복 제출 작업 흐름은 멱등성, 상태 가드, 비활성 행동, 충돌 처리를 가진다.
- 공개 토큰, 저장 경로, 비공개 URL, 내부 ID, 민감 필드는 프로필이 명시 허용하지 않는 한 노출하지 않는다.

## UX와 접근성

- 대상 런타임과 입력 방식에 맞춰 설계한다.
- 런타임이 요구하면 주요 행동은 한 손/터치 사용을 지원한다.
- 로딩, 빈 상태, 오류, 재시도, 인증 필요, 권한 없음, 오래된 상태, 오프라인, 권한 거부 상태를 명시한다.
- 사용자 표시 문구는 프로젝트의 i18n/content 시스템을 따른다.
- 포커스 상태, 스크린 리더 라벨, 움직임 줄이기, 안전 영역, 텍스트 넘침을 확인한다.

## 런타임

- 앱 재개, deep link, push, 보안 저장소, 권한 흐름이 중복 내비게이션이나 오래된 상태를 만들지 않는다.
- 런타임이 지원하면 브라우저 미리보기 또는 대체 동작을 정의한다.
- 권한 거부는 정상 사용자 상태로 처리한다.
- 프로덕션 빌드에서 목/데모 override로 API나 권한 경계를 우회하지 않는다.

## 검증

- 관련 단위/통합 테스트 또는 대상 확인을 실행한다.
- 컴파일된 프론트엔드 코드에 영향을 주면 빌드/타입체크를 실행한다.
- 동작 변경에는 RED/GREEN/REFACTOR 증거가 있거나 문서화된 예외가 있다.
- 런타임별 변경은 플랫폼/브라우저 검증 또는 잔여 위험이 포함된 명시적 수동 검증을 남긴다.
- 백엔드 계약 변경은 `docs/harness/context/integration/api-matrix.md`를 갱신하고 통합 리뷰를 통과한다.
