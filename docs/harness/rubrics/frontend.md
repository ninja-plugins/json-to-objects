# 주요 프론트엔드 리뷰 기준

> 프로젝트별 행위자/API/토큰/패키지 값은 `docs/harness/profiles/**`에 둔다.

주요 프론트엔드 상세 기준은 `docs/harness/02_PRIMARY_FRONTEND.md`에 있다. 이 파일은 기존 참조 호환을 위한 얇은 체크리스트다.

## 체크리스트

- 기존 로컬 컴포넌트와 스타일 토큰을 새 변형보다 먼저 사용했다.
- 디자인 추가/리팩토링은 `docs/harness/07_DESIGN_SYSTEM.md`와 디자인 시스템 프로필을 따른다.
- UI는 프로젝트가 소유한 스타일을 사용하며 승인 없는 외부 스타일/UI 프레임워크가 없다.
- 프로젝트의 명시 예외가 없다면 인라인 스타일이 없다.
- 새 사용자 표시 문구는 프로젝트의 i18n/content 시스템을 따른다.
- 반응형 동작, 터치 영역, 키보드/포커스 상태, 텍스트 넘침을 확인했다.
- API 구조/인증/리소스/페이지네이션 변경은 필요한 경우 백엔드 짝 검토와 통합 리뷰를 포함했다.
- 동작 변경에는 RED/GREEN/REFACTOR 증거가 있거나 문서화된 예외가 있다.
- 변경 위험에 맞는 프론트엔드 테스트/빌드 또는 브라우저 확인을 실행했다.
- 장기 사실이 바뀌었다면 관련 프론트엔드/통합 컨텍스트를 갱신했다.
